# Split-Flap Card v0.1.0-beta.14

Security and release metadata hotfix following the `v0.1.0-beta.13` pre-release.

## Summary

This beta focuses on closing the GitHub CodeQL warnings for DOM text being reinterpreted as HTML and preparing the project for continued beta work without carrying an open security warning.

## Security

- Fixed CodeQL alert path where preview summary text could be reinterpreted as HTML.
- Fixed CodeQL alert path where diagnostics output used `innerHTML`.
- Changed diagnostics rendering to use plain text via `textContent`.
- Preserved diagnostics information while avoiding raw HTML injection for dynamic text.

## Fixed

- Fixed `_previewSummary()` dynamic text handling.
- Fixed diagnostics rendering sink from `innerHTML` to `textContent`.
- Fixed diagnostics output so it returns plain multiline text instead of HTML markup.

## Changed

- Updated package metadata to `0.1.0-beta.14`.
- Clarified the beta release path after `v0.1.0-beta.13`.
- Kept this release intentionally small so feature development can continue cleanly in the next beta.

## Notes

- This is a pre-release / beta hotfix.
- Refresh Home Assistant dashboard resources after updating.
- Clear frontend/browser cache if the old frontend file remains loaded.
- Advanced animation work such as upper-flap fall, over-roll and more realistic mechanical motion remains planned for the next feature beta.
