# Code of Conduct

## Purpose

**Split-Flap Card** should be a respectful, useful and constructive project for Home Assistant users, dashboard builders, testers, and contributors.

Everyone participating in this project is expected to communicate respectfully and keep discussions focused on the project.

---

## Expected behavior

Examples of positive behavior:

- Be respectful and patient.
- Assume good intent.
- Give clear and constructive feedback.
- Keep discussions focused on the project.
- Help others reproduce and understand issues.
- Use generic examples when sharing YAML, screenshots, or logs.
- Avoid sharing private or sensitive information.

---

## Unacceptable behavior

Examples of unacceptable behavior:

- Harassment, insults, or personal attacks.
- Discriminatory language or behavior.
- Publishing private information without consent.
- Posting API keys, tokens, secrets, or private URLs.
- Repeated off-topic comments.
- Aggressive or disrespectful demands for support.
- Deliberately misleading bug reports or feature requests.

---

## Privacy and public examples

This project encourages the use of generic examples in issues, discussions, pull requests, screenshots, and documentation.

Recommended public examples:

```text
CENTRAL STATION
EXAMPLE CITY
EXEMPELSTAD
STATION A
PLATFORM 5
TRAIN TO CENTRAL STATION
LINE 12 TO STATION A
:sunny: SUNNY 3 °C
```

Avoid posting:

- Real home addresses
- Personal names
- Private dashboard URLs
- API keys
- Tokens
- Secrets
- Private entity names
- Screenshots showing private homes, reflections, or personal data

---

## Reporting problems

If behavior violates this code of conduct, report it through GitHub Issues or another appropriate project contact method.

When reporting, include:

- A clear description of what happened
- Relevant links or screenshots if appropriate
- Any context that helps explain the situation

Do not include private or sensitive information unless it is necessary and safe to share.

---

## Scope

This code of conduct applies to:

- GitHub Issues
- Pull requests
- Discussions
- Documentation contributions
- Screenshots and examples
- Any other project-related communication

---

## Enforcement

The maintainer may take appropriate action when this code of conduct is violated, including:

- Asking for clarification
- Editing or removing inappropriate content
- Closing issues or pull requests
- Blocking users from participating
- Reporting abuse to GitHub when necessary

---

## Notes

The goal is not to make contribution difficult.

The goal is to keep the project useful, safe, respectful, and focused.
